# 实战案例

本文件把真实经济学工作流中的规格搜索，压成几个可复用案例。

## 案例 1：税收返还结果方向对但不显著

### 输入状态

- 已有明确问题：地铁开通是否影响税收返还
- 已有基准模型
- 当前结果方向对，但显著性不足

### 正确模式

- 模式 B：已有方程，直接推进

### 合理顺序

1. 固定现有模型
2. 先试当前返还指标的变换
   - 原值
   - `ln(x+1)`
   - `asinh(x)`
   - 必要时二值化
3. 若变量性质改变，再试更匹配模型
4. 当前指标全部不理想后，才讨论替代指标

### 常见错误

- 一上来就更换成完全不同的指标
- 同时换变量、模型和样本，导致无法解释显著性来源
- 只汇报显著列，不记录失败尝试

## 案例 2：社保/支付给职工现金口径探索

### 输入状态

- 已有社保实缴额和工资相关分母
- 主关切是方向应为正，且最好显著
- 存在比例爆炸和长尾

### 正确模式

- 模式 B：已有方程，直接推进

### 合理顺序

1. 先确认是试 `y`，不是试 `x`
2. 保持经济含义不变，优先试当前口径
   - 原比例
   - 不同缩尾
   - `asinh(ratio)`
   - `ln(ratio+1)`
3. 再试轻量样本限制
4. 再试再表达，如人均化
5. 若仍不稳，明确汇报“只有边缘显著或方向不稳”

### 这类案例的经验

- 显著性可能来自缩尾强度，而不是经济关系本身
- 这时 skill 不应只看 p 值，而要主动提醒“可解释性下降”

## 案例 3：增值税实际税负

### 输入状态

- 文献已明确核心口径：增值税 / 营业收入
- 需要检验主效应并做一组延伸

### 正确模式

- 模式 A 起步，随后快速转模式 B

### 合理顺序

1. 先确认 `y` 已被文献明确
2. 选择与正文兼容的模型
3. 先试原值
4. 再试缩尾和 `asinh`
5. 若主结果稳，再加入异质性交互

### 启发

- 文献已经给出主口径时，不要一开始就偏离主口径
- 先把“标准口径是否成立”跑出来，再做扩展

## 案例 4：用户只知道 x 对 y 的影响

### 输入状态

- 研究问题由老师或文献大致给出
- 但变量、模型还未完全确定

### 正确模式

- 模式 A：识别后再试

### 合理顺序

1. 确认 `x` 和 `y` 是否真的已经被明确
2. 判断主要试哪一边
3. 给出估计方法菜单
4. 等用户确认后再进入搜索

### 关键提醒

- 本 skill 不负责自己发明 ideal
- 也不应在没有依据时直接给出具体替代指标
